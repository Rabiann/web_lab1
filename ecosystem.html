Now, let's write simple http-client in Rust

Adding dependencies

First off all, let's add reqwest crate to our Cargo.toml.


                    cargo add reqwest

Taking input paraneters

Let's write logic for entering link via CLI parameter


                    fn main() {
                        let params: Vec = std::env::args().into_iter().collect();
                        let link = params[1]; // first param is the name of file
                        println!("Link: {}", link);
                    }

Now we can run a program

cargo run -- google.com

And get a result

Link: google.com

Actually writing a client

Now lets write client which will return response body


                    fn main() {
                        let params: Vec = std::env::args().into_iter().collect();
                        let link = params[1];
                        println!("Link: {}", link);

                        let response = reqwest::blocking::get(&link).unwrap();

                        println!("{:?}", response.text());
                    }

After executing we will get response body

What about Rust's ecosystem?

The Rust ecosystem is still relatively young and evolving, especially compared to more established languages like C++ and Python. However, it is fast-growing and gaining momentum due to strong community support, contributions from major tech companies, and its unique focus on performance and safety.

Where can I get libraries for my project?

Cosider visiting crates.io. It is an official repository of Rust so known crates.

Where I can read documentation for them?

Documentation for all crates is placed on docs.rs. There you can find docs, modules, cargo-features and source code. Some crates and frameworks can have their own websites, like actix_web.
